# 双线车道线检测 — 算法文档

---

## 一、算法管线总览

```
灰度图像 (GRAYSCALE, 400×240, 90fps)
  │
  ├─ ① 直方图二值化       OTSU 自适应阈值分割，地面=黑(0)，车道线=白(255)
  ├─ ② 底部种子点搜索      从图像最底行中点向两侧找黑白交界
  ├─ ③ 8邻域边缘追踪      左右两侧各沿 8 邻域模板向上追踪车道线边缘
  ├─ ④ 中线点计算         逐行取左右边缘中点
  ├─ ⑤ 最小二乘直线拟合   x = k·y + b（y 归一化避免病态）
  ├─ ⑥ 计算 offset/angle
  └─ ⑦ 丢线防抖           连续 5 帧无效才上报丢线
```

---

## 二、各步骤算法详述

### 2.1 直方图二值化

| 项目 | 说明 |
|------|------|
| 算法 | OTSU 大津法（`img.get_histogram().get_threshold()`） |
| 输入 | 传感器直出灰度图（GRAYSCALE，0~255） |
| 输出 | 二值图（0 或 255） |
| 目的 | 自适应分离地面（暗）和车道线（亮），不依赖固定阈值 |

**为什么用 OTSU 而非固定阈值**：场地光照变化大，固定阈值在强光下过曝、弱光下漏检。OTSU 根据当前帧直方图自动计算最优分割点，每帧自适应。

---

### 2.2 底部种子点搜索

从图像最底行（`y = h-1`）的中间列出发，寻找左右两侧的黑白交界作为追踪起点：

```
如果中点像素 > 128（白色，车在车道内）:
    向左扫描找连续两黑 → 左车道线内边缘
    向右扫描找连续两黑 → 右车道线内边缘

如果中点像素 < 128（黑色，压在线上）:
    向两侧交替扫描找白色区域
    → 定位到白区边缘 → 再向外找到黑白交界
```

种子点分别作为左追踪和右追踪的起点。

---

### 2.3 8邻域边缘追踪（核心算法）

#### 2.3.1 基本思想

从底部种子点出发，逐行向上追踪。每步在当前点 8 邻域中寻找"黑白交界"——一侧像素值 < 128（黑），另一侧 > 128（白）。取 y 坐标最小的候选点作为新位置，保证追踪方向向上。

#### 2.3.2 邻域模板

```
左边缘追踪（顺时针）:          右边缘追踪（逆时针）:
  (-1,0) (-1,-1) (0,-1)         (1,0) (1,-1) (0,-1)
  (-1,1)   P     (1,-1)         (-1,1)  P    (-1,-1)
  (0,1)  (1,1)  (1,0)          (0,1) (1,1) (-1,0)
```

两者方向相反是因为左边缘的白区在右、黑区在左，右边缘相反。

#### 2.3.3 追踪终止条件

| 条件 | 含义 |
|------|------|
| 连续 3 步位置不变 | 已到车道线尽头或图像顶部 |
| 左右追踪点收敛（|x_l - x_r| < 2 且 |y_l - y_r| < 2） | 两条车道线交汇（消失点） |
| 达到最大迭代次数（h × 2/3） | 安全上限，防止死循环 |

---

### 2.4 中线点计算

对左右追踪点按行配对，取中点：

```
mid_x = (left_x + right_x) // 2
mid_y = (left_y + right_y) // 2
```

得到中线点集 $P_M = \{(x_k, y_k)\}$。这些点代表车道中心线的像素坐标。

---

### 2.5 最小二乘直线拟合

#### 2.5.1 模型

$$x = k \cdot y + b$$

- $k$：斜率（$dx/dy$），反映车道线相对摄像头的偏转
- $b$：截距（$y = 0$ 处的 x 位置）

**为什么用直线而非二次曲线**：追踪点覆盖高度有限（~160 行），短距离内直线近似已足够。二次拟合在点数少时容易过拟合。

#### 2.5.2 y 归一化

y 值范围 0~239，直接拟合会导致矩阵病态（$y^2$ 达 57121，行列式计算精度丢失）。解法：**所有 y 值先除以 y_max 归一化到 [0, 1]**，拟合后再反归一化。

$$y_{\text{norm}} = \frac{y}{y_{\max}}$$

$$x = k_n \cdot y_{\text{norm}} + b_n$$

反归一化：

$$k = \frac{k_n}{y_{\max}}, \quad b = b_n$$

---

### 2.6 计算横向偏移与航向偏差

#### 2.6.1 横向偏移 (offset)

在图像底部（离车最近处）计算：

$$x_{\text{mid}} = k \cdot (h - 1) + b$$

$$\Delta x_{\text{px}} = x_{\text{mid}} - \frac{W}{2}$$

$$\text{offset}_{\text{mm}} = \Delta x_{\text{px}} \cdot \alpha$$

其中 $\alpha$ = `MM_PER_PIXEL`（mm/px，需场地标定）。

**符号约定**：

| offset | 含义 | M0 响应 |
|--------|------|---------|
| > 0 | 车道右偏，车偏左 | 右打方向 |
| < 0 | 车道左偏，车偏右 | 左打方向 |

#### 2.6.2 航向偏差 (angle)

拟合直线 $x = k y + b$ 中，$k = dx/dy$ 是车道线在图像中的斜率。

$$\theta = \arctan(k)$$

| k | θ | 含义 |
|---|-----|------|
| > 0 | 正 | 中线向右倾斜，车头偏左 |
| = 0 | 0 | 中线竖直，车头正对车道 |
| < 0 | 负 | 中线向左倾斜，车头偏右 |

---

### 2.7 丢线防抖

| 项目 | 说明 |
|------|------|
| 算法 | 计数器 + 滞后阈值 |
| 参数 | MAX_CONSECUTIVE_INVALID = 5 |

90fps 下 5 帧 ≈ 55ms，过滤单帧噪点，真正丢线时快速响应。

## 三、参数汇总

| 参数 | 默认值 | 单位 | 说明 |
|------|--------|------|------|
| IMAGE_WIDTH | 400 | px | 图像宽度 |
| IMAGE_HEIGHT | 240 | px | 图像高度 |
| FPS | 90 | Hz | 帧率 |
| SENSOR_ID | 2 | - | 摄像头 ID |
| FIT_MIN_POINTS | 8 | 个 | 拟合最少点数 |
| MM_PER_PIXEL | 1.25 | mm/px | 像素到 mm 比例 |
| MAX_CONSECUTIVE_INVALID | 5 | 帧 | 连续丢线阈值 |
| SEARCH_DEPTH_RATIO | 2/3 | - | 追踪深度占图像高度比例 |

## 四、算法复杂度

| 步骤 | 时间复杂度 | 说明 |
|------|-----------|------|
| 直方图二值化 | O(W·H) | C 层实现，极快 |
| 种子点搜索 | O(W) | 单行扫描 |
| 8邻域追踪 | O(H × W) 最坏, O(H) 典型 | 实际追踪点数远小于全图 |
| 中线点计算 | O(H) | |
| 直线拟合 | O(n) | n ≈ 追踪点对数量 |

总体 O(W·H)，但常数极小。400×240 @ 90fps 在 K230 上可轻松满足。

## 五、现场调参指南

### 5.1 调参顺序

```
第一步：让车能检测到线
  观察 OpenMV IDE 视觉效果：
    - 二值图车道线清晰连续 → 没问题
    - 车道线碎/断 → 调整摄像头角度，使车道线更清晰
    - 噪声太多 → 检查场地光线，避免强反光

第二步：让 offset 方向对
  车故意偏左放 → offset 应为正 → 正确
  车故意偏左放 → offset 为负 → MM_PER_PIXEL 取负值

第三步：让 angle 方向对
  车头故意偏左 → angle 应为正 → 正确
  车头故意偏左 → angle 为负 → 公式加负号

第四步：跑直道测试
  车在直道上自主跑 2m：
    走直线不扭 → 正常
    来回摇摆   → M0 侧 Kp 过大
    偏向一侧   → offset 零点偏差
```

### 5.2 常见问题

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 不追踪 | 二值化后车道线不明显 | 检查 IDE 二值图，调整摄像头角度/曝光 |
| 追踪到一半停止 | 车道线在某处断开 | 检查场地车道线是否连续 |
| offset 飘移 | MM_PER_PIXEL 不准 | 重新量测标定 |
| 帧率低于 90fps | 追踪循环过长 | 减小 SEARCH_DEPTH_RATIO |
