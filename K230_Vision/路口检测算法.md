# 路口检测 — 算法文档

---

## 一、问题描述

比赛路口的特征：车道上**突然出现一条与车道方向垂直的黑色标线**。

```
正常直道                    路口
┌─────────────┐           ┌─────────────┐
│   ║     ║   │           │   ║     ║   │
│   ║     ║   │           │   ║     ║   │
│   ║     ║   │           │─────────────│ ← 横向黑线
│   ║     ║   │           │   ║     ║   │
│   ║     ║   │           │   ║     ║   │
└─────────────┘           └─────────────┘
 全是灰白路面               远场出现黑色横条
```

关键洞察：路口黑线只出现在**两条车道线之间**，不会延伸到图像最左/最右。所以检测框只需要放在中间有效区域。

---

## 二、算法管线

```
二值化图像 (0=黑, 255=白)
  │
  ├─ ① 确定检测区域    左右各留 80px 边距, 距顶 40px, 高 12px
  ├─ ② 均匀放置 6 个框  等分有效区域
  ├─ ③ 统计黑像素      每框黑像素占比 ≥ 50% → 框标记为 1
  ├─ ④ 找最长连续 1 段
  ├─ ⑤ 类型判定        十字 / T字左 / T字右
  └─ ⑥ 多帧确认        连续 3 帧同类型 → 上报
```

---

## 三、框布局

```
     0              80                   320           400
  0  ┌──────────────┬────────────────────┬──────────────┐
     │   跳过(80)   │  ← 有效区域(240)→  │   跳过(80)   │
     │              │  ┌──┬──┬──┬──┬──┬──┐              │
 40  │              │  │0 │1 │2 │3 │4 │5 │              │
     │              │  └──┴──┴──┴──┴──┴──┘              │
 52  │              │   0  0  1  1  1  0   ← 标 1/0    │
     │              │   INT: CROSS                       │
     └──────────────┴────────────────────┴──────────────┘
```

左右各留 80px 边距，中间 240px 等分 6 框（每框 40px 宽）。

---

## 四、类型判定

| 条件 | 类型 | direction |
|------|------|-----------|
| 最长连续 1 < 3 框 | 无路口 (0) | — |
| ≥ 70% 框都为 1 | 十字 (1) | 直行 (3) |
| 左侧框的 1 多于右侧 | T字左 (2) | 左转 (1) |
| 右侧框的 1 多于左侧 | T字右 (3) | 右转 (2) |

## 五、参数汇总

| 参数 | 默认值 | 说明 |
|------|--------|------|
| NUM_BOXES | 6 | 检测框数量 |
| BOX_LEFT_MARGIN | 80 px | 左边缘跳过 |
| BOX_RIGHT_MARGIN | 80 px | 右边缘跳过 |
| BOX_HEIGHT | 12 px | 每框高度 |
| BOX_Y_OFFSET | 40 px | 框排距图像顶部 |
| BLACK_FILL_RATIO | 0.5 | 黑像素占比阈值 |
| MIN_ACTIVE_BOXES | 3 | 最少连续活动框数 |
| CONFIRM_FRAMES | 3 | 多帧确认帧数 |

## 六、线程模型

路口检测在 **K230 小核 (800MHz)** 独立线程运行，与大核 (1.6GHz) 的巡线+显示并行。

```
主线程 (大核)                      路口线程 (小核)
  snapshot()                            │
  lane.Process(img) ← 原地二值化        │
  lane.SendToUart()                     │
  img.copy() ──共享──→ _inter_img       │
  _inter_ready = True                   ├─ _inter_ready? → 取 _inter_img
  draw 叠加层                           ├─ inter.Process(copy, [], [])
  Display.show_image()                  ├─ inter.SendToUart()
                                        └─ _inter_ready = False
```

- 共享变量 `_inter_img` + `_inter_ready` 标志，无需锁
- 主线程拷贝帧后立即返回，不阻塞
- 路口线程处理上一帧副本，不影响当前帧显示

---

## 七、性能

每帧路口检测仅遍历 6×(40×12) = 2880 像素，耗时 < 1ms。加上独立线程运行，对主线程帧率零影响。
